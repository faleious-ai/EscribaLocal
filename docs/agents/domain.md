# Domain Docs

Este repositório é de contexto único.

## O que ler

- `CONTEXT.md` na raiz do repositório.
- `docs/adr/` para decisões arquiteturais relevantes.
- `docs/` para documentação de domínio relacionada ao tema em análise.

## Como consumir

- Leia primeiro `CONTEXT.md` antes de planejar ou alterar comportamento não trivial.
- Leia ADRs que toquem a área em questão.
- Se `CONTEXT.md` ou `docs/adr/` não existirem, prossiga sem bloquear o trabalho.

## Estrutura esperada

```text
/
├── CONTEXT.md
├── docs/adr/
└── src/
```
